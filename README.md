## About Me

I am a Software Engineer and AI/ML Specialist dedicated to engineering high-performance, production-grade intelligent systems and scalable software architectures. Currently pursuing my Bachelor of Science in Software Engineering at the Ghulam Ishaq Khan Institute (GIKI), I bridge the gap between rigorous research-grade artificial intelligence and enterprise-ready software engineering.

My technical experience spans the complete lifecycle of intelligent applications: from training and optimizing deep learning architectures in PyTorch and TensorFlow, to converting and deploying models via highly optimized ONNX inference pipelines on CPU-only, memory-constrained cloud infrastructures. In the software engineering domain, I architect and build robust, secure backend systems utilizing Spring Boot, Node.js, Express, and Flask, integrating them with responsive React frontends and clean PostgreSQL/NoSQL databases.

Driven by strong computer science fundamentals—spanning Data Structures, Analysis of Algorithms, Software Design & Architecture, and Operating Systems—I prioritize clean architecture, design patterns (Strategy, Factory, Observer), and performance-focused code. I am actively expanding my capabilities into generative AI and large language model systems, specializing in prompt engineering, retrieval-augmented generation (RAG), and advanced fine-tuning methodologies.

I am deeply interested in exploring and researching advanced frontiers in machine learning, specifically Vision Transformers, Mixture of Experts (MoE), Quantum Machine Learning, Agentic AI, and distributed systems. My ultimate goal is to build secure, explainable, and highly scalable AI systems that deliver tangible real-world impact.

**Open To:** Graduate Research, AI/ML Engineering, Computer Vision Research, Backend/Software Engineering, and Core Open Source Contributions.

### Explainable Multimodal Medical Diagnosis System
An end-to-end explainable multimodal deep learning system fusing chest X-ray imagery (CNN) and clinical text notes (BERT) for clinical diagnosis.

* **Architecture:** Dual-tower network architecture. Image features are extracted via a Convolutional Neural Network (CNN) backbone. Textual notes are processed using a pre-trained BERT transformer to capture semantic clinical context. Cross-modal late fusion aligns and merges the text and visual feature representations, feeding them into a classification layer.
* **Key Features:** Late-fusion pipeline, image-text alignment, clinical token embeddings, F1/ROC-AUC evaluation metrics.
* **Engineering Challenges:** Resolving cross-modal representation gaps and handling severe diagnostic class imbalances in training medical datasets. Implemented weighted sample loss formulations to ensure generalizability.
* **Impact:** Integrated Grad-CAM (visual feature heatmaps) and token attention highlights into a web UI, giving doctors clean visual explanations behind deep learning predictions.

| Stack | Performance | Security | Architecture | Impact |
| :--- | :--- | :--- | :--- | :--- |
| PyTorch, BERT, CNN, OpenCV, Flask | ~88% ROC-AUC on testing set | Modality inputs validation | Dual-Tower Late Fusion | High clinical explainability via Grad-CAM |

### Looks Maximizer - AI-Powered Facial Analysis & Recommendation System
A production facial structure analyzer and styling recommendation application deploying multi-model ONNX pipelines onto lightweight serverless infrastructure.

* **Architecture:** Decoupled architecture with a React-based frontend and a Python/ONNX inference server backend.
* **Key Features:** Multi-model pipeline orchestration, PyTorch-to-ONNX serialization, styling recommendation generation, low-latency CPU inference.
* **Engineering Challenges:** Deploying and serving three concurrent CNN models on CPU-only, memory-constrained environments (Railway starter tier) without incurring high costs or container crashes. Resolved this by converting PyTorch weights to optimized, quantised ONNX models and writing custom stateless process stabilization controllers.
* **Impact:** Under 3-second user inference latency with zero GPU requirements, saving over 65% in production hosting costs.

| Stack | Performance | Security | Architecture | Impact |
| :--- | :--- | :--- | :--- | :--- |
| PyTorch, ONNX, Railway, Vercel, React | < 3s Inference Latency on CPU | Rate limiting / Input scrubbing | Decoupled client-server | Deployed multi-model ONNX on serverless |

### MindMate - AI-Integrated Mental Wellness Platform
A full-stack wellness hub featuring automated facial emotion recognition to deliver personalized mood insights and wellness tracking.

* **Architecture:** Decoupled service architecture. Backend business logic and role-based access controls are handled by Spring Boot. Real-time facial emotion recognition (FER-2013) is delegated to a Python microservice.
* **Key Features:** Real-time FER pipeline, JWT authentication, user wellness diary, database-backed history log.
* **Engineering Challenges:** Managing cross-origin API orchestration, handling state synchronizations, and establishing low-latency cross-service network boundaries. Used Strategy, Factory, and Observer OOP design patterns to decouple business layers.
* **Impact:** Clean, highly modular structure permitting independent deployment of the Python AI service and the Java REST API.
